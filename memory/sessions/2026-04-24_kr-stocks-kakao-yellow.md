### 1. 코스피/코스닥 주식 지원 (커밋 62c4d58)
- `detect_market(symbol)`: 6자리 숫자 = KR, 영문 = US
- `normalize_symbol()`: US는 대문자, KR은 원형 유지
- **Yahoo Finance `v8/finance/chart`** API로 한국주식 실시간 시세 조회 (`.KS` → `.KQ` 순 폴백)
- 60초 인메모리 캐시, User-Agent 헤더 필수
- `StockHolding.current_price` 컬럼 추가 (KR 시세 API 실패 시 수동 폴백용). ALTER TABLE 패턴.
- `/api/admin/stock` POST/PUT: 심볼 형식 검증(영문만 OR 6자리 숫자만)
- `/api/assets`: 종목별 `market`/`currency`/`cost_krw`/`gain_krw`/`gain_percent`, 전체 `total_cost_krw`/`total_gain_krw`/`total_gain_percent` 추가
- 관리자 폼: 시장 선택 드롭다운(나스닥/코스피/코스닥) + 구매단가 입력 + 폴백 현재가(선택) — 시장에 따라 placeholder/힌트 동적 변경

### 2. UI 리디자인 — 화이트 + 옐로우 테마
- 커밋 62c4d58: 머스터드 옐로우(#CA8A04)로 전환 → 사용자 "너무 진하다" 피드백

- 옐로우 배경 위 텍스트는 **검정(#191600, `--primary-ink`)** 으로 통일 (카카오 로고 스타일, 가독성)
- `--dark-hero`: 차콜 그라데이션 (#0F172A → #334155) — 벌금카드/이벤트히어로/공지바 배경
- **자산 요약 카드 리디자인**: 화이트 배경으로 전환. 평가금액(22px 두꺼운 폰트) + 수익률(+%/+원, 한국식 빨/파) + 총투자금/현금보유 메타 라인
- 종목 리스트: 시장 배지 + 내 평균 + 수익률 뱃지(pill) + 평가액. 한국식 등락 컬러
- **Pretendard Variable** 폰트 CDN 적용, `letter-spacing: -0.01em`
- 보라색 흔적 24개 RGBA + 7개 hex 전부 제거
- PWA 아이콘 8개 2회 재생성 (머스터드 → 카카오 옐로우)

### 3. 종목명 표시 — 한글 회사명 (커밋 65ae227)
- 사용자 피드백: "005930 보이지 말고 '삼성전자'로 표시"
- **네이버 증권 모바일 API** 발견: `https://m.stock.naver.com/api/stock/{code}/basic`
  - 응답에 `stockName: "삼성전자"` 한글 포함. No API key. No rate limit 체감.
  - Yahoo의 `shortName`은 영문 약어("SamsungElec")이거나 깨진 값(`247540.KS,0P0001GZPV,...`) 반환 — 불안정
- KR: 네이버 우선 → Yahoo 폴백
- US: Finnhub `stock/profile2` → Yahoo 폴백
- `StockHolding.name` 컬럼 추가 (ALTER TABLE). 주식 추가 시 즉시 조회하여 저장. `/api/assets`에서 기존 데이터 레이지 백필.
- 회사명 6시간 인메모리 캐시(`_name_cache`)
- UI: 종목명 우선 표시, 원본 심볼은 `stock-code` 클래스로 작은 회색 보조 텍스트

## 교훈 (다음 세션 참고)

- 한국 주식 관련 기능은 **네이버 m.stock API가 가장 신뢰성 높음** — 글로벌 API 먼저 시도하다 시간 낭비 말 것.
- 옐로우 배경에 흰 글씨는 **절대 금지** — 가독성 제로. 카카오 스타일 검정 텍스트 규칙을 따를 것.
- ALTER TABLE + try/except 패턴은 sqlite/postgres 모두에서 잘 작동하지만, 프로덕션 첫 실행 로그는 반드시 확인.
- Windows 콘솔에서 한글 테스트 시 `sys.stdout.reconfigure(encoding='utf-8')` 또는 hex 덤프로 검증.
- Bash에서 `UID` 변수명 readonly — 다른 이름 사용.
